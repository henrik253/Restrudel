# analysis/results — machine-readable corpus statistics

JSON exported by [`../../notebooks/01_corpus_analysis.ipynb`](../../notebooks/01_corpus_analysis.ipynb)
for reuse by `data_gen/`. Regenerated by re-running the notebook.

## Common envelope
Every file carries the same top-level keys:

```json
{
  "schema_version": 1,
  "generated_from": "notebooks/01_corpus_analysis.ipynb",
  "n_patterns": 855,
  "n_sources": 8,
  "schema": "<name>.v1",
  ...payload...
}
```

`weight` = share of total occurrences (sums to ~1 within a file); `prob` = a
conditional probability. Both are ready for direct weighted sampling.

## Files

| file | payload | use in generation |
|---|---|---|
| `sounds.json` | `items[]`: `{name, occurrences, patterns, category, weight}` | pick a sound by `weight` |
| `functions.json` | `items[]`: `{name, occurrences, patterns, category, weight}` | pick transforms/effects |
| `banks.json` | `items[]`: `{name, occurrences, weight}` | pick a drum-machine bank |
| `mini_notation.json` | `items[]`: `{feature, patterns, fraction}` | how often to use `[]`,`<>`,euclid,… |
| `sound_categories.json` | `items[]`: `{category, occurrences, weight}` | synth vs sample mix |
| `complexity.json` | `voices`, `functions_per_pattern` (describe stats), `patterns_with_tempo` | target realistic size/polyphony |
| `transitions.json` | `n_chains`, `top_bigrams[]`, `depth1{}`, `depth2{}` | **Markov chain** for method order |

### transitions.json
```json
{
  "depth1": {
    "s": { "total": 1383,
           "successors": [ {"name": "gain", "count": 329, "prob": 0.238}, ... ] }
  },
  "depth2": {
    "s": { "total": 521,
           "successors": [ {"path": ["gain","room"], "count": 31, "prob": 0.059}, ... ] }
  }
}
```
To build a chain: start from a head (`s`/`note`/`n`/`stack`), then repeatedly
sample the next call from `depth1[current].successors` by `prob` (use `depth2`
for 2-step lookahead). This reproduces idiomatic chains like
`note → s → lpf → room → gain`.
